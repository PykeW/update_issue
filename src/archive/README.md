# 归档模块说明

## 概述

本文档说明已归档到 `src/archive/` 目录的模块，这些模块在当前版本中未被使用，但保留以备将来可能的需求。

## 归档模块列表

### 1. auto_sync_manager.py
- **功能**: 自动同步管理器
- **归档原因**: 当前系统使用API触发的立即同步机制，不再需要后台自动同步服务
- **可能用途**: 如果将来需要恢复后台自动同步功能，可以参考此模块

### 2. change_detector.py
- **功能**: 变更检测器，检测数据库变更并自动触发同步
- **归档原因**: 当前系统通过API直接处理同步，不需要变更检测机制
- **可能用途**: 如果将来需要实现数据库变更监听功能，可以参考此模块

### 3. optimized_issue_creator.py
- **功能**: 优化议题创建器
- **归档原因**: 功能已整合到 `gitlab_operations.py` 中
- **可能用途**: 如果将来需要独立的议题创建优化功能，可以参考此模块

### 4. progress_monitor.py
- **功能**: 进度监控器，监控GitLab议题进度变化
- **归档原因**: 当前系统专注于数据同步，不需要进度监控功能
- **可能用途**: 如果将来需要实现进度监控功能，可以参考此模块

### 5. smart_queue_processor.py
- **功能**: 智能队列处理器，高效处理同步队列
- **归档原因**: 当前系统使用简化的队列处理机制，不需要复杂的智能处理
- **可能用途**: 如果将来需要实现高级队列处理功能，可以参考此模块

## 恢复指南

如果需要恢复某个归档模块：

1. **评估需求**: 确认是否真的需要该模块的功能
2. **检查依赖**: 确保相关依赖和配置仍然可用
3. **更新导入**: 修改导入路径以匹配新的项目结构
4. **测试功能**: 充分测试恢复的模块功能
5. **更新文档**: 更新相关文档以反映模块的恢复

## 注意事项

- 归档模块可能包含过时的代码或配置
- 恢复前请仔细检查代码的兼容性
- 建议在恢复前创建备份
- 恢复后需要更新相关的测试和文档
